How agent actions become a tamper-evident audit trail

A 30-second replay of the real demo run: three ADK agents investigate an incident; every action is captured by callbacks and sealed into that agent's own signed hash chain each publish cycle — and any log you hand the verifier is proven authentic, or caught.

The components (and where each lives)

ADK agents

research_demo/agent.py

Three LlmAgents collaborating in parallel on the same incident, sharing session state. Each is an independent audit principal with its own identity and key.

Callbacks

research_demo/action_logger.py

ADK's six hooks (before/after agent, model, tool) fire our logger: one record per action, written to the per-agent JSONL log and handed to the audit spool.

Spool

research_demo/audit_spool.py

The pending queue between live agents and the trails — one spool file per agent (<agent>.pending.jsonl), each rotated atomically at publish.

Publisher

audit_publisher.py

Fires every minute (demo: 15s). Rotates each agent's spool and invokes auditlog ingest per trail — with agents acting constantly, every anchored entry is a batch.

auditlog ingest

halfprice/AuditLog

Signs each action (ed25519), extends the hash chain (prev_record_hash, monotonic seq), builds a Merkle root, and seals a signed manifest for the period.

Walrus + cloud storage

halfprice/AuditLog · src/backends

Raw signed records go to write-once storage; the small manifest goes to Walrus, where the blob id is itself a content digest — the published commitment.

Sui & Walrus anchor

halfprice/AuditLog · move/audit_trail

Every agent has its own trail: each manifest blob on Walrus has a matching Sui object, blobs are prev-linked into a per-agent chain, and a head object always points at that chain's tail — exactly one successor admitted per height. (How entries are attested before confirmation is an implementation detail, elided here.)

Verify endpoint

verify_server.py → auditlog verify-log

POST any log file: signatures are checked against the on-chain key, the Merkle root is recomputed from the uploaded bytes, and any edit, forgery, or fabrication is pinpointed by seq.

Run the real thing: ./demo.sh --interval 15 --keep-serving — then curl --data-binary @exported_agent_0.jsonl http://127.0.0.1:8600/verify. This page replays a recorded run; timings compressed, content verbatim.
