Why this matters

Two different threats

A rogue agent hiding its actions is one problem. A dishonest operator rewriting history is another — and it's the harder one. This handles both.

Logging lives outside the agent

Each action is forwarded to an independent collector the instant it happens. A hijacked agent can't delete what it can't reach.

Write-once isn't proof to outsiders

WORM storage stops insiders, but the operator owns the policy and can reset it. “We locked it” is not something a regulator can verify.

The chain is public

Digests are anchored to Walrus + Sui, which the operator doesn't control and can't rewrite. That's the part no cloud config can fake.

Anyone can verify

A third party — auditor, regulator, counterparty — re-derives the digests from public sources, trusting neither the operator nor the cloud.

The honest limit

The collector must sit on the agent's mandatory path — an egress proxy it can't skip. An agent that bypasses logging entirely can still act unseen; no audit system escapes that.
